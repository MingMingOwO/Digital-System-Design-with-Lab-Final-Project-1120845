## 1. 專題名稱
基於 RISC-V 核心之環境感測製作一台自動溫控風扇

## 2. 使用開發板
Basys 3 開發板

## 3. 使用工具版本
* **硬體開發工具：** Vivado (2026.1)
* **軟體編譯工具：** RISC-V Toolchain

## 4. 專案資料夾結構
```text
├── hw/                     # FPGA 硬體設計電路與 Vivado 工程檔案
│   ├── rtl/                # Verilog 原始碼 (PicoRV32, 自製 PWM, I2C 封裝等)
│   ├── constrs/            # Basys 3 開發板腳位約束檔 (Basys3.xdc)
│   └── testbench/          # 模擬驗證用的 Testbench 檔案
├── sw/                     # 運行於 RISC-V 核心之嵌入式軟體原始碼
│   ├── main.c              # 主程式邏輯 (溫度輪詢、轉速控制、LCD 顯示)
│   └── Makefile            # 自動化編譯與記憶體檔案生成腳本
└── README.md               # 本專案說明文件
```

## 5. 如何產生 bitstream
1. 開啟 Vivado 軟體，並載入 `hw/` 目錄下的專案檔（或透過 Tcl 腳本重建專案）。
2. 確保 `hw/rtl/` 資料夾下所有自製與開源之 Verilog 模組已正確加入設計層級。
3. 於 Flow Navigator 點擊 Run Synthesis 與 Run Implementation。
4. 確認無誤後點擊 Generate Bitstream，等待系統生成最終的 `.bit` 檔案。

## 6. 如何載入或修改 RISC-V 程式
1. 進入 `sw/` 資料夾，開啟 `main.c` 以修改溫度判定門檻（預設為 28 度）或優化控制邏輯。
2. 於終端機執行 `make` 指令，透過 RISC-V 交叉編譯器將 C 語言程式碼編譯為機器碼。
3. 腳本會自動將機器碼轉換為 Vivado 可辨識的記憶體初始化檔案（如 `.mem` 或 `.hex` 檔）。
4. 將產生的記憶體檔案更新至 Instruction/Data Memory 模組中，以供硬體合成時載入。

## 7. 如何燒錄到 FPGA 開發板
1. 使用 micro-USB 傳輸線將 Basys 3 開發板與主機電腦連接，並開啟板上電源開關。
2. 在 Vivado 中開啟 Hardware Manager 並點擊 Auto Connect 以識別 FPGA 晶片。
3. 右鍵點擊目標晶片選擇 Program Device。
4. 載入先前生成好的位元流檔案（`.bit`），點擊 Program 開始燒錄。

## 8. 如何操作與測試
### 硬體週邊接線配置
* **I2C 匯流排 0 (Bus 0)：**將 DHT11 溫濕度感測器與 LCD1602 顯示螢幕同時掛載於系統的 I2C SCL 與 SDA 腳位上。
* **PWM 訊號輸出：**將指定的 Pmod 輸出腳位連接至馬達驅動板，用以調控直流風扇馬達電力。
* **獨立供電：**為避免板載電源不穩，風扇馬達之驅動電力需接入獨立外部電源。

### 系統測試與驗證步驟
* **常溫觀測：**系統上電後，確認 LCD1602 是否正確顯示當前常溫數字，且風扇保持低速運轉。
* **升溫測試：**使用吹風機小心靠近 DHT11 進行加熱，當偵測溫度超過 28°C 門檻時，驗證風扇是否立即切換至強風模式，且 LCD 同步更新狀態。
* **降溫測試：**使用冰塊接近感測器，確認溫度回落至 28°C 以下時，風扇是否自動回復原先的低速或平滑運轉。
* **全域重置：**按下開發板上的 Reset 按鈕，確認 LCD 螢幕清空重開，且風扇轉速瞬間歸零。
* **斷線防呆：**故意拔除 DHT11 的資料線，確認系統並未鎖死或當機，且 LCD 螢幕能夠跳出 ERR 錯誤代碼。

## 9. 已知問題
* **板載電力限制：**若直接利用 Basys 3 開發板的輸出電力推動實體直流馬達，在高負載下會引發供電不穩進而導致 CPU 核心重置。目前採取的解決備案為透過外部獨立電力驅動馬達，FPGA 僅輸出 PWM 控制訊號。
